데이터를 시각적으로 표현하는 방법을 구체적으로 설계합니다.

1.  `docs/prd.md`, `docs/dataflow.md`, `docs/api.md` 문서를 읽고, 사용자가 확인해야 할 주요 지표와 프론트엔드에서 사용할 수 있는 데이터 형식을 파악합니다.
2.  대시보드에 표시될 모든 시각화 컴포넌트(차트, 그래프 등)를 목록화합니다.
3.  각 컴포넌트에 대해 다음 항목을 상세하게 명세합니다.
    *   **컴포넌트 ID:** 고유 식별자 (예: `monthly-performance-line-chart`)
    *   **제목:** 차트의 제목 (예: "월별 실적 추이")
    *   **차트 유형:** `Line Chart`, `Bar Chart`, `Pie Chart` 등
    *   **데이터 소스:** 데이터를 가져올 API 엔드포인트와 응답 데이터 내의 특정 필드
    *   **데이터 매핑:** X축, Y축, 범례(Legend) 등에 어떤 데이터 필드를 매핑할지 명확하게 정의합니다.
4.  완성된 시각화 명세를 `docs/dataviz-spec.md` 경로에 저장합니다.

**반드시 다음 규칙을 준수해야 합니다:**

-   각 시각화 컴포넌트는 사용자가 얻어야 할 인사이트가 무엇인지 명확히 드러내야 합니다.
-   데이터 매핑은 프론트엔드 개발자가 혼동 없이 즉시 구현할 수 있을 정도로 구체적이어야 합니다.
-   `prd.md`에서 정의된 비즈니스 목표와 직접적으로 관련 없는 시각화는 포함하지 않습니다.
