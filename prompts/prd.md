## 2. 사용자 스토리 (User Stories)
> "As a [사용자 역할], I want [수행할 행동], so that [달성하려는 목표]" 형식으로 MVP의 핵심 기능들을 기술합니다.

> **⚠️ 중요:** 이 템플릿을 사용할 때는 반드시 프로젝트의 실제 데이터 복잡도를 고려해야 합니다. 본 프로젝트는 4개의 CSV 파일(department_kpi.csv, publication_list.csv, research_project_data.csv, student_roster.csv)과 각 파일당 수십 개의 컬럼을 포함하는 복잡한 데이터 구조를 다루므로, 이러한 복잡도를 반영한 사용자 스토리를 작성해야 합니다.

- **US-001: 로그인**
  - As a 관리자, I want to 로그인할 수 있다, so that 허가된 사용자만 시스템에 접근할 수 있다.
- **US-002: 데이터 업로드**
  - As a 관리자, I want to CSV 파일을 업로드할 수 있다, so that 시스템이 4개 유형의 데이터(학과 KPI, 논문, 연구과제, 학생 명부)를 분석할 수 있다.
- **US-003: 대시보드 조회**
  - As a 관리자, I want to 주요 지표가 시각화된 대시보드를 볼 수 있다, so that 데이터 기반의 의사결정을 신속하게 할 수 있다.
- **US-004: 학과별 KPI 조회**
  - As a 관리자, I want to 학과별 연도별 주요 성과 지표(취업률, 교원 수, 기술이전 수입액, 국제학술대회 개최 횟수 등)를 조회할 수 있다, so that 각 학과의 성과를 비교하고 평가할 수 있다.
- **US-005: 논문 게재 현황 조회**
  - As a 관리자, I want to 학과별/저널등급별 논문 게재 현황을 조회할 수 있다, so that 연구 실적을 파악하고 SCIE, KCI 등 저널 등급별 현황을 확인할 수 있다.
- **US-006: 연구비 집행 내역 조회**
  - As a 관리자, I want to 연구과제별 예산 집행 내역 및 집행률을 조회할 수 있다, so that 연구비 사용 현황을 모니터링하고 예산 관리가 적절히 이루어지는지 확인할 수 있다.

## 3. 핵심 기능 목록 (Feature List)
> 각 사용자 스토리를 구현하기 위한 구체적인 기능들을 나열합니다.

- **F-01: 사용자 인증**
  - 아이디/비밀번호 입력 폼
  - 로그인 요청 처리 및 세션/토큰 발급
- **F-02: CSV 파일 업로드**
  - 파일 선택/드래그앤드롭 UI (4개 파일 유형 지원: 학과 KPI, 논문, 연구과제, 학생 명부)
  - 파일 형식 유효성 검사 (클라이언트 사이드)
  - 파일 전송 API
- **F-03: 데이터 파싱 및 저장**
  - 업로드된 CSV 데이터 파싱 로직 (4개 파일 유형별 파싱 처리)
  - 파싱된 데이터 유효성 검사 (서버 사이드, 각 파일별 수십 개 컬럼에 대한 복잡한 검증 규칙)
  - DB에 데이터 저장 (각 파일별 테이블 구조 반영)
- **F-04: 대시보드 시각화**
  - 주요 지표 데이터 조회 API
  - 막대, 라인 차트 등 시각화 컴포넌트 렌더링
- **F-05: 학과별 KPI 조회 기능**
  - 학과별/연도별 필터링 기능
  - 취업률, 교원 수, 기술이전 수입액, 국제학술대회 개최 횟수 등 지표 조회 API
  - 비교 차트 시각화 (학과 간 비교, 연도별 추이)
- **F-06: 논문 게재 현황 조회 기능**
  - 학과별/저널등급별 필터링 기능
  - SCIE, KCI, 일반 저널 분류별 통계 조회 API
  - 논문 목록 및 통계 차트 시각화
- **F-07: 연구비 집행 내역 조회 기능**
  - 과제번호별/학과별/상태별 필터링 기능
  - 예산 집행률 계산 및 집행 내역 조회 API
  - 집행 현황 차트 및 상세 내역 테이블 시각화

## 4. 범위 외 (Out of Scope)
> 이번 MVP 개발 범위에 포함되지 않는 항목을 명확히 하여 혼선을 방지합니다.

- 사용자별 권한 관리
- Ecount 시스템과 직접 연동
- 상세 데이터 필터링 및 검색 기능
