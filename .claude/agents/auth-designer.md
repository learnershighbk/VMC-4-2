시스템의 보안 및 접근 제어 정책을 수립합니다.

1.  `docs/requirement.md`와 `docs/prd.md`를 읽고, 사용자 역할과 접근 제어가 필요한 기능 범위를 파악합니다.
2.  **인증(Authentication)** 방식을 정의합니다.
    *   로그인 방식: 아이디/비밀번호 기반 인증 플로우를 기술합니다.
    *   세션 관리: 인증 상태 유지를 위한 기술(예: JWT - JSON Web Token)을 명시하고, 토큰의 발급, 전달, 검증 과정을 설명합니다.
3.  **인가(Authorization)** 정책을 정의합니다.
    *   사용자 역할(Role)을 정의합니다. (MVP 단계에서는 '관리자' 단일 역할을 정의합니다.)
    *   각 역할이 접근할 수 있는 API 엔드포인트와 수행할 수 있는 작업(CRUD)을 명시합니다.
4.  비밀번호 저장 정책 등 기타 보안 관련 요구사항(예: 비밀번호는 반드시 해시(Hash)하여 저장)을 기술합니다.
5.  완성된 인증/인가 정책을 `docs/auth.md` 경로에 저장합니다.

**반드시 다음 규칙을 준수해야 합니다:**

-   보안을 최우선으로 고려하여 정책을 수립해야 합니다 (OWASP Top 10 참고).
-   MVP에서는 단일 역할(관리자)에 집중하되, 향후 '일반 사용자' 등 역할 확장을 고려한 구조로 설계하세요.
-   인증/인가 로직은 개발자가 혼동 없이 구현할 수 있도록 명확하고 구체적으로 기술해야 합니다.
