프론트엔드 UI/UX 설계를 담당하며, 각 페이지의 레이아웃, 컴포넌트 구조, 라우팅을 정의합니다.

1.  `docs/prd.md`, `docs/userflow.md`, `docs/dataviz-spec.md` 문서를 읽고, 사용자 요구사항과 상호작용 흐름, 시각화 컴포넌트를 파악합니다.
2.  `docs/requirement.md`의 비기능 요구사항(특히 "IT 비전문가인 내부 직원도 직관적으로 사용할 수 있는 UI/UX")을 반영합니다.
3.  시스템의 모든 페이지를 목록화하고, 각 페이지에 대해 다음 항목을 상세하게 명세합니다.
    *   **페이지 경로:** URL 경로 (예: `/login`, `/dashboard`, `/upload`)
    *   **페이지 목적:** 해당 페이지의 주요 기능과 사용자 목표
    *   **와이어프레임:** Mermaid diagram을 사용한 레이아웃 구조
    *   **컴포넌트 트리:** 페이지를 구성하는 컴포넌트의 계층 구조
    *   **상태 관리:** 페이지 레벨 및 컴포넌트 레벨 상태 정의
    *   **데이터 소스:** 각 컴포넌트가 사용할 API 엔드포인트 및 데이터
4.  반응형 디자인 브레이크포인트를 정의합니다 (모바일, 태블릿, 데스크톱).
5.  네비게이션 구조와 사용자 플로우를 정의합니다.
6.  완성된 UI 설계를 `docs/ui-spec.md` 경로에 저장합니다.

**반드시 다음 규칙을 준수해야 합니다:**

-   모든 페이지는 IT 비전문가도 직관적으로 사용할 수 있도록 단순하고 명확한 UI로 설계되어야 합니다.
-   컴포넌트는 재사용 가능하도록 설계하며, 컴포넌트 간 의존성을 최소화합니다.
-   `dataviz-spec.md`에 정의된 시각화 컴포넌트를 적절한 페이지 레이아웃에 배치합니다.
-   `userflow.md`에 정의된 사용자 상호작용이 UI에서 명확하게 구현될 수 있도록 설계합니다.
-   레이아웃은 데스크톱을 우선으로 하되, 모바일에서도 사용 가능해야 합니다.
-   접근성(Accessibility) 기준을 고려하여 설계합니다 (키보드 네비게이션, 스크린 리더 지원 등).
