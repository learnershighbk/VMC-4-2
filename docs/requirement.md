## 1. 프로젝트 개요

- **프로젝트명:** 대학교 내부 데이터 시각화 대시보드 개발
- **핵심 목표:** Ecount 시스템에서 추출한 CSV 데이터를 기반으로, 주요 성과 지표(실적, 논문, 학생 수, 예산 등)를 직관적으로 파악할 수 있는 웹 대시보드를 구축한다.
- **대상 사용자:** 대학교 내부 임직원

## 2. 프로젝트 배경 및 목표

- **현황 (As-Is):**
    - Ecount 시스템의 기본 그래프는 시인성과 활용도가 낮아 데이터 기반의 신속한 의사결정에 어려움이 있음.
    - 주요 성과 데이터가 CSV 파일 형태로 관리되어 취합 및 보고에 수동 작업이 요구됨.

- **목표 (To-Be):**
    - **핵심 목표:** 데이터 기반 정보를 시각적으로 쉽고 편리하게 확인하여, 신속하고 정확한 의사결정을 지원한다.
    - **부가 목표:** 수동 데이터 취합 및 보고 과정을 자동화하여 업무 효율을 개선한다.

## 3. 과업 범위 (Scope of Work)

### 3.1. 포함 범위 (In-Scope)

- 웹 기반 대시보드 프론트엔드 개발
- 데이터 처리 및 API를 위한 백엔드 시스템 개발
- 사용자 인증 기능 구현
- CSV 데이터 업로드, 파싱 및 DB 저장 기능 구현

### 3.2. 미포함 범위 (Out-of-Scope)

- Ecount 시스템과의 직접적인 API 연동 (CSV 업로드 방식 기반)
- 사용자별/부서별 상세 권한 관리 기능 (1차 MVP에서는 단일 권한)
- 데이터 분석 및 리포트 자동 생성 기능

## 4. 주요 기능 요구 사항 (Functional Requirements)

- **R1: 데이터 업로드:** 관리자는 Ecount에서 다운로드한 CSV 파일을 웹 대시보드에 업로드할 수 있어야 한다.
- **R2: 데이터 처리:** 시스템은 업로드된 CSV 데이터를 자동으로 파싱하여 지정된 데이터베이스 스키마에 맞게 저장해야 한다.
- **R3: 데이터 시각화:** 사용자는 대시보드에서 주요 지표(실적, 논문, 학생 수, 예산 등)를 막대, 라인, 파이 차트 등 다양한 형태의 그래프로 확인할 수 있어야 한다.
- **R4: 사용자 인증:** 허가된 사용자는 아이디와 비밀번호를 통해 시스템에 로그인할 수 있어야 한다.

## 5. 비기능 요구 사항 (Non-functional Requirements)

- **사용성:** IT 비전문가인 내부 직원도 직관적으로 사용할 수 있는 UI/UX를 제공해야 한다.
- **성능:** 대시보드 데이터 조회 및 로딩은 3초 이내에 완료되어야 한다.
- **확장성:** 향후 새로운 데이터 지표나 시각화 차트 유형을 쉽게 추가할 수 있는 구조여야 한다.
- **보안:** 허가되지 않은 사용자는 민감한 데이터에 절대 접근할 수 없어야 한다.
