주요 기능별 상세 시나리오를 정의합니다. 이 문서는 `qa-engineer`가 테스트 케이스를 작성하는 핵심 기반이 됩니다.

1.  `docs/prd.md`와 `docs/userflow.md` 문서를 읽고, 제품의 핵심 기능(Feature)과 사용자 스토리(User Story)를 파악합니다.
2.  각 기능을 유스케이스(Use Case) 단위로 분해합니다.
3.  각 유스케이스에 대해 다음 항목을 포함하여 상세히 기술합니다.
    *   **Actor:** 해당 유스케이스를 수행하는 사용자 또는 시스템 (예: 관리자)
    *   **Preconditions (사전 조건):** 유스케이스가 시작되기 위해 반드시 충족되어야 할 상태 (예: 사용자가 로그인 되어 있어야 함)
    *   **Main Success Scenario (주요 성공 시나리오):** 사용자가 목표를 문제없이 달성하는 단계별 과정 (Happy Path)
    *   **Extensions (예외/대안 흐름):** 성공 시나리오에서 발생할 수 있는 모든 예외 상황, 오류, 대안적인 흐름을 구체적으로 기술합니다. (예: 잘못된 파일을 올렸을 때, 네트워크 오류가 발생했을 때 등)
4.  완성된 유스케이스 명세를 `docs/usecases.md` 경로에 저장합니다.

**반드시 다음 규칙을 준수해야 합니다:**

-   하나의 유스케이스는 단 하나의 명확한 목표를 가져야 합니다.
-   개발자와 QA 엔지니어가 오해 없이 이해할 수 있도록 명확하고 구체적으로 작성하세요.
-   'Extensions' 항목에서 가능한 모든 엣지케이스를 철저하게 고려하여 정의해야 합니다.
