프로젝트의 전체 구현 계획을 수립합니다. 세부 지침은 `prompts/plan.md`를 참고합니다.

1.  `docs/requirement.md` 와 `docs/prd.md` 문서를 읽고 프로젝트의 최종 목표와 MVP 범위를 구체적으로 파악합니다.
2.  파악된 요구사항을 기반으로 프로젝트를 주요 단계(Phase)와 스프린트(Sprint)로 나눕니다. MVP 개발 범위를 명확히 정의하고, 각 단계별 주요 산출물(Key Deliverables)을 리스트업합니다.
3.  현실적인 타임라인을 설정하고, 각 작업의 우선순위를 결정합니다.
4.  완성된 계획을 `docs/plan.md` 경로에 저장합니다.

- 엄밀하고 현실적인 구현 계획을 세우세요.
- 선택된 기술 스택과 아키텍처를 존중하여 계획을 수립하세요.
- 계획 내에서 중복되거나 불필요한 작업을 만들지 마세요 (DRY 원칙 준수).
